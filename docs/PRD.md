# LovTrans - 旅游翻译 PRD

## 1. 产品定位

**一句话描述**：为中国出境游用户打造的 AI 翻译工具，提供口语化翻译和旅游场景专属体验。

**核心差异化**：
- LLM 驱动的口语化翻译（vs Google/DeepL 的书面翻译）
- 3语言快切设计（母语 + 目的地语言 + 英语）
- 旅游场景专属 UX（大字体展示、一键切换）

## 2. 目标用户

**MVP 用户**：中国用户出境游（日本、泰国、韩国、欧美等）

**用户场景**：
- 餐厅点餐时与服务员沟通
- 问路、打车时交流
- 购物时询价、砍价
- 酒店入住、求助时沟通

## 3. MVP 功能清单

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 文字翻译 | 输入文字 → LLM 口语化翻译 → 显示结果 |
| P0 | 语音输入 | 语音 → 火山ASR → 文字 → 翻译 |
| P0 | 语音播放 | 翻译结果 → 火山TTS → 播放 |
| P0 | 3语言设置 | 设置母语/目的地语言/通用语言 |
| P0 | 智能语言识别 | 自动检测输入是哪种语言 |
| P1 | 用户账户 | 注册/登录（复用现有 Supabase Auth） |
| P1 | 订阅管理 | 订阅制付费（复用现有 Pricing 组件） |
| P2 | 历史记录 | 保存翻译历史，方便复用 |

**未来功能（不在 MVP）**：
- 图片翻译（菜单OCR）
- 实时对话模式
- 离线翻译

## 5. 技术方案

| 组件 | 方案 |
|------|------|
| 框架 | Next.js 15 (基于现有 lovtrans) |
| LLM | ZenMux 多模型切换 |
| ASR | 火山引擎语音识别 |
| TTS | 火山引擎语音合成 |
| Auth | Supabase Auth（现有） |
| DB | Supabase + DrizzleORM（现有） |
| 部署 | Vercel + PWA |
